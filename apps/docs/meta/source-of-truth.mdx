This page is the maintenance map for `apps/docs`.

## Why this exists

Memory Crystal has source material spread across code, scripts, READMEs, and internal design docs.
The public docs should not guess. They should point back to the exact repo files that own the truth.

## Page depth tiers

### Tier A — flagship pages
These pages should always include:
- **Plain-language summary**
- **What this means in practice**
- **How it actually works**
- **Commands / examples**
- **Source of truth**

Tier A pages for the first implementation batch:
- `introduction`
- `quickstart`
- `integrations/mcp-server`
- `integrations/openclaw-plugin`
- `tools/overview`
- `api/authentication`
- `organic/overview`
- `configuration/environment-variables`

### Tier B — support/reference pages
These pages should include:
- concise explanation
- practical usage notes
- examples when relevant
- source links

### Tier C — internal/private source material
These pages are **not** public docs.
They can inform public docs, but they should not be mirrored directly.

## Freshness triggers

Update `apps/docs` when any of these change:

- install scripts under `apps/web/public/`
- `packages/mcp-server/README.md` or `mcp-server/README.md`
- plugin config fields or plugin wiring files
- `convex/crystal/mcp.ts` or `convex/crystal/knowledgeHttp.ts`
- `.env.example`
- `shared/tierLimits.ts`
- `convex/crystal/organic/**` or `convex/crons.ts`
- brand assets or design tokens used by the docs shell

## Public mirror rule

The public mirror should expose public docs from `apps/docs`.
Internal markdown in `docs/` and agent instruction files should stay out of the public mirror.
